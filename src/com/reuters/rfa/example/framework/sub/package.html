Contains a framework for a consumer/subscriber application. SubAppContext is the main component of this framework. It performs tasks that most consumer/subscriber needs to do before it can consume/subscribe data.

Configurable parameters are
  • debug - enable debug tracing. The default is false.
  • serviceName - service to request. The default is "".
  • runTime - how long the application should run before exiting (in seconds). The default is -1 (infinite).
  • session - session name to use. The default is "myNamespace::mySession".
  • user - DACS username for login. The default is the value from "user.name" property or "guest".
  • position - DACS position for login. The default is local host or "1.1.1.1/net".
  • application - DACS application ID for login. The default is "256".
  • fileDictionary - load dictionary from file. The default is false.
  • enumType - enumtype.def filename. The default is "/var/triarch/enumtype.def".
  • type - type of subscriber connection. It can be either "OMM" or "MarketData". The default is "OMM".

    • The following parameter can be used if "type" is "OMM".

    • rdmFieldDictionary - RDMFieldDictionary filename. The default is "/var/triarch/RDMFieldDictionary".
    • requestPARSupport - Encode login request with "SupportPauseResume = 1" . The default is "True".

    The following parameters can be used if "type" is "MarketData".

    • mounttpi - use TokenizedPrincipalIdentity for Session. The default is false.
    • appendixaFilename - appendix_a filename. The default is "/var/triarch/appendix_a".
