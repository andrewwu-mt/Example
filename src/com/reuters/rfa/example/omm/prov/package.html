StarterProvider_Interactive is a simple provider example to demonstrate how to publish Reuters Domain Model data using the Open Message Model. This application is server application One thread dispatches client session connection request, item requests, and timer events. Updates are sent when the timer event is received. A consumer application (source distributor, P2PS or RFA OMMConsumer) may connect to the server to request data. Currently, the StarterProvider_Interactive application provides Logins, Service Directories, Dictionaries, and Market Price. It can be easily extended to provide other message model types. StarterProvider_Interactive supports pause and resume, consumer can send a Login request that contain SupportPauseResume entry in the attrib. StarterProvider_Interactive will respond with the same value as the request. If SupportPauseResume does not exist in the Login request, it will also be eliminated from the response. However no update conflation logic implemented.

StarterProvider_Interactive provides examples of how to:
  • Create a Session and an EventQueue
  • Create an OMMProvider event source
  • Listen to client connections
  • Handle simple pause resume request
  • Dispatch events from an EventQueue
  • Cleanup a Session

The StarterProvider_Interactive application currently supports the following command line parameters.

  • debug - enable detailed logging for RFA (default is "false")
  • provSession - name of Provider session. (default is "myNamespace::provSession")
  • serviceName - Service name for the SrcDirectory response. (default is "DIRECT_FEED")
  • listenerName - Unique name that specifies a connection to listen. (default is """(null)")
  • updateInterval - Update interval in seconds. (default is "1")
  • updateRate - Update rate per interval. (default is "2")
  • rdmFieldDictionary - RDM Field dictionary name and location.  (default is "/var/triarch/RDMFieldDictionary")
  • enumType - Enum dictionary name and location.  (default is "/var/triarch/enumtype.def")
  • runTime - How long (in seconds) application should run before exiting (default is "600")

The example can be run as an application with the following commands:

java com.reuters.rfa.example.omm.prov.StarterProvider_Interactive -serviceName DIRECT_FEED
