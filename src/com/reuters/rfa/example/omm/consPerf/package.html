StarterConsumer_Perf demonstrates basic principles of RFA Java development using OMM. It is written to show a simple way of developing a consumer type application which is focused on performance. More elaborate and or specialized applications may be written based on this example.

For simplicity reasons this application does a limited error checking during initialization and exit time.

This application handles only the Level 1 streaming data, MarketPrice domain. It requests a number of items, all streaming level 1 data.

It shows a simple way of requesting, consuming and displaying Level 1 data as well as it provides statistics showing an approximate image and update rates that it is able to handle. This application does not measure latency of any item.

This application allows for configuration of session name, usage of event queue, time interval between stats displays, etc.

Basic behavior of this application is as follows:

  1. Initialize application's and RFA's specific components
  2. Register timer event to display statistics at displayInterval
  3. Request login
  4. Wait for successful login response
  5. Request items
  6. Handle incoming Level 1 data either in RFA session thread (if nullEQ is true) or in application's Dispatcher thread
    • Decode and display all items if printData flag is set
    • Store statistics information for refresh,updates and status messages. Other type of messages are ignored.
  7. Depending on the configuration display incoming data statistic and / or content
  8. Exit after specified run time

The OMMSimpleConsumerPerf application currently supports the following command line parameters.

  • runTime- Run time of the test application. Defaults to 600 secs.
  • session- Consumer session. Defaults to myNamespace::mySession
  • serviceName- Service name to request. Defaults to DIRECT_FEED
  • itemName- List of items to open separated by ','. Defaults to TRI.N
  • nullEQ- Test with null EventQueue. Defaults to false
  • rdmFieldDictionary- RDMField dictionary name and location. Defaults to /var/rdm/RDMFieldDictionary
  • enumType- RDMEnum dictionary name and location. Defaults to /var/rdm/enumtype.def
  • displayInterval- Throughput display rate (in seconds). Defaults to 5
  • printData- Flag to print update response
  • printStatistics- Flag to print statistics

The example can be run as an application with the following commands:

java com.reuters.rfa.example.omm.consPerf.StarterConsumer_Perf -serviceName DIRECT_FEED -itemName "A.ARC" -session RSSLNamespace::localConsumerSession -runTime 600 -nullEQ false -displayInterval 5 -printData false

Application exits itself after specified run time
