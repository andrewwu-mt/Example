This example aims to demonstrate how to publish Reuters Domain Model data using the Open Message Model. The DomainServer is the main class of provider application which uses PubAppContext, the provide framework, to publish the Open Message Model data.

Currently, this application provides Logins, Service Directories, and Dictionaries data as default. It can also provide Market Price, Market By Order, Market By Price, Market Maker, Symbol List data. If needed, you can easily extend the example to provide other types of data.

This application:

  • Reads the input parameter from the command line argument.
  • Provides OMM data for basic domains such as: Login, Directory and Dictionary by using {@linkplain com.reuters.rfa.example.framework.prov ProviderFramework}.
  • Provides the other domains based on commandline argument such as: Market Price, Market By Order, Market By Price, Market Maker, and Symbol List

The DomainServer application currently supports the following command line parameters.

  • debug: Enables debug tracing. The default is false.
  • provSession: Sets the session name to use. The default is myNamespace::provSession.
  • rdmFieldDictionary: Sets the RDMFieldDictionary path and filename. The default is /var/triarch/RDMFieldDictionary.
  • enumType: Sets the enumtype.def filename. The default is /var/triarch/enumtype.def.
  • listenenerName: Sets the listener name for the session. The default is "".
  • acceptSessions: Sets whether to accept all sessions. The default is true.
  • vendor: Sets the vendor's name. The default is RFAExample.
  • pubServiceName: Sets the name of the service used by provider. The default is MY_SERVICE.
  • isSource: Sets whether this is the original source of data. The default is false.
  • runTime: Sets the duration (in seconds) for which the application should run before exiting. The default is -1.
  • mmt: List of comma-delimited domains supported by the server. The default is MARKET_PRICE,MARKET_BY_ORDER,MARKET_BY_PRICE,MARKET_MAKER,SYMBOL_LIST.
  • <domain name>_updateInterval example:MARKET_BY_ORDER_updateInterval: Sets the interval (in seconds) between each update message. The default is 3.
  • <domain name>_encodeDataDef example:MARKET_BY_ORDER_encodeDataDef: Enables the option to optimize bandwidth by using DataDefinitions. The default is true.
  • SYMBOL_LIST_totalSymbols: Specifies the total number of symbols in the Symbol List. The default is 25.

The example can be run as an application with the following commands:

    java com.reuters.rfa.example.domainServer.DomainServer -provSession myNameSpace::myProviderSession -mmt MARKET_BY_ORDER -pubServiceName DIRECT_FEED
