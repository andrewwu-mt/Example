The StarterConsumer_SymbolList program demonstrates the use of RFA Java OMM in subscribing to a SYMBOL_LIST and then making non-streaming requests for all of the items in the SYMBOL_LIST. The items in the SYMBOL_LIST can be requested in any of the following message model types: {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_PRICE MARKET_PRICE}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_BY_ORDER MARKET_BY_ORDER}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_BY_PRICE MARKET_BY_PRICE}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_MAKER MARKET_MAKER}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#SYMBOL_LIST SYMBOL_LIST}.

This program uses the utility object SubAppContext for context initialization.

This example uses the following command line configuration parameters:

  • help: Displays this message on screen. The default is false.
  • session: Sets the consumer session. The default is myNamespace::mySession.
  • serviceName: Sets the service name for the SrcDirectory response.
  • itemName: Sets the item name for consumer requests. The default is 0#ARCA.
  • debug: Sets the application to print out debug messages. The default is false.
  • runTime: Sets the length of time (in seconds) for which the application runs before exiting. The default is -1.
  • mmt: Sets the message model type string. The default is MARKET_PRICE.
  • type: Sets the type of subscriber connection. It can be either OMM or MarketData. For this example, it must be OMM (which is its default setting). 

You can run the StarterConsumer_SymbolList using the command:

    java com.reuters.rfa.example.omm.symbollist.StarterConsumer_SymbolList -itemName 0#ARCA -serviceName DF_ARCA
