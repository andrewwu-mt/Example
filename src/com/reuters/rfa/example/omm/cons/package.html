StarterConsumer demonstrates how to subscribe to Reuters Domain Model data using RFA's OMM Consumer model. It can connect to an RFA OMMProvider / P2PS / RDFD using RSSL connection.

This application does the following:
  • Register for Login
  • Handle Login response
  • Register for item(s) request with one of the following models: {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_PRICE MARKET_PRICE}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_BY_ORDER MARKET_BY_ORDER}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_BY_PRICE MARKET_BY_PRICE}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_MAKER MARKET_MAKER}, {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#SYMBOL_LIST SYMBOL_LIST}
  • Use {@link com.reuters.rfa.example.utility.GenericOMMParser GenericOMMParser} to parse {@link com.reuters.rfa.omm.OMMMsg OMMMsg}

The StarterConsumer application currently supports the following command line arguments.

  • debug - enable detailed logging for RFA (default is "false")
  • session - name of consumer session to acquire. Its connection list should include at least one RSSL connection.  (default is "myNamespace::mySession")
  • serviceName - name of source directory to use. (default is "DIRECT_FEED")
  • itemName - List of items to open separated by ','. (default is "TRI.N") Please note that ',' is a valid character for RIC name. The application need to be modified if RIC names have ','.
  • mmt - Message Model Type. (default is "MARKET_PRICE")
  • attribInfoInUpdates - Ask provider to send OMMAttribInfo with update and status messages (default is "false")
  • user - DACS username for Login (default is value of  user.name system property)
  • position - DACS position for login for Login (default is IP address/hostname)
  • application - DACS application ID for Login (default is "256")
  • rdmFieldDictionary - RDM Field dictionary name and location.  (default is "/var/triarch/RDMFieldDictionary")
  • enumType - Enum dictionary name and location.  (default is "/var/triarch/enumtype.def")
  • runTime - How long (in seconds) application should run before exiting (default is "600")

The example can be run as an application with the following commands:

java com.reuters.rfa.example.omm.cons.StarterConsumer -itemName CSCO.O -serviceName DIRECT_FEED

Note: An OMM consumer application doesn't need to make a request for directory. As a result this example doesn't request for DIRECTORY. User need to know a service name that is available.
