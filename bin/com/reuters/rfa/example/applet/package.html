The purpose of this applet example is to demonstrate how RFA Java can be used as an applet application. An applet example for consumer side is {@linkplain com.reuters.rfa.example.omm.gui.quotelist QuoteList}

Before running the applet, the environment needs to be set up and source codes need to be complied and packaged to Jar as follows:

Setting up environment:
Running an applet in a browser requires the Java(TM) Plug-in, which is installed by default with the JDK and JRE. In order to run RFA Java on an applet environment, following security permissions need to be enabled in the applet runtime

  • permission java.util.PropertyPermission "com.reuters.rfa.validPackages", "read";
  • permission java.lang.RuntimePermission "modifyThreadGroup";
  • permission java.lang.RuntimePermission "modifyThread";
  • permission java.util.logging.LoggingPermission "control";
  • permission java.io.FilePermission "<log file name>.lck", "write"; (for logging to file.)
  • permission java.io.FilePermission "<log file name>", "write"; (for logging to file.)
  • permission java.net.SocketPermission "<ip address>:<port>" , "connect";
  • permission java.net.SocketPermission "<ip address>:<port>" , "resolve";
However, instead of manually changing permission, the jar file of applet classes can be signed in order to get full permission access. Please follow the section below to build the singed jar.

Compiling and packaging source code and signing applet Jar files:
 1. Set up an appropriate JAVA_HOME environment variable.
 2. Run buildApplet.bat (for Windows) or buildApplet.ksh (for Unix and Linux) in /Examples/applet folder.
The script will build a jar file (rfaj_applet_example.jar) that contains the example classes. After that the rfaj.jar and the example jar file will be signed. An applet_example.ks file is a keystore file (JKS format) which contains a key that is used to sign the jar files.
After running the script successfully, you will see rfaj_applet_example.jar and signed_rfa.jar in the same path of the build script.

Running QuoteList applet:
Open quotelist.html on the browser that installed java plug-in, accept the certification then the applet will appear.
More details see {@linkplain com.reuters.rfa.example.omm.gui.quotelist QuoteList}.
