The ItemGroupsDemo example demonstrates how to manage item groups in a consumer application. ItemGroupsDemo currently supports the following command line configuration parameters:

  • session: Sets the name of the consumer session to acquire. Its connection list should include at least one RSSL connection. By default this is: myNamespace::mySession.
  • serviceName: Sets the name of the source directory to use. By default, this is DIRECT_FEED.
  • itemName: Sets the comma-delimited list of items to open. The default setting is TRI.N.

    • Note: The comma (,) is a valid character for a RIC name. You must modify the application if any RIC names include a comma (,).
  • mmt: Sets the Message Model Type. By default, this is MARKET_PRICE.
  • attribInfoInUpdates: Sets the provider to send OMMAttribInfo with update and status messages. By default this is false.
  • user: Sets the DACS username for Login. By default, this is the value of the user.name system property.
  • position: Sets the DACS position for Login. By default, this is the IP address/hostname.
  • application: Sets the DACS application ID for Login. By default, this is 256.
  • rdmFieldDictionary: Sets the RDM Field dictionary name and location. By default, this is /var/triarch/RDMFieldDictionary.
  • enumType: Sets the enum dictionary name and location. By default, this is /var/triarch/enumtype.def.
  • runTime: Sets the length in time (in seconds) the application runs before exiting. By default, this is 600.

The example can be run as an application with the following commands:

    java com.reuters.rfa.example.omm.itemgroups.ItemGroupsDemo -itemName CSCO.O -serviceName DIRECT_FEED
