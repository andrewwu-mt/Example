StarterProvider_SourceMirroring is a simple interactive server provider example that demonstrates how to receive generic messages with the SourceMirroringMode.

Package: com.reuters.rfa.example.omm.sourcemirroringprov

Source code location: $RFA/Examples/com/reuters/rfa/example/omm/sourcemirroringprov

StarterProvider_SourceMirroring provides examples of how to:

  • Provide Login, Directory, and Generic Messages on item streams
  • Include AcceptingCustomerStatus in the Directory response
  • Receive and process generic messages with Consumer Status Mode
  • Receive and process generic messages from a consumer application
Running the Example

This example uses the following command line parameters:

  • debug: Enables debug tracing. The default is false.
  • provsession: Specifies the name of the session to acquire. The default is RSSLNamespace::localProviderSession.
  • runTime: Sets the number of seconds for which the program runs before exiting. The default is 600.
  • submitInterval: Sets the update interval (in seconds). The default is 1.
  • acceptingConsumerStatus: Sets whether the provider can handle Consumer Status Generic Messages. The default is 1.

You can run this example as an application with the following command:

    java com.reuters.rfa.example.omm.sourcemirroringprov.StarterProvider_SourceMirroring -acceptingConsumerStatus 1
