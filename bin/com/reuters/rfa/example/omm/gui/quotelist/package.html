The QuoteList is a GUI application that uses RFA to retrieve several fields each from a list of instruments (real-time OMM level 1 data) which update in real time. The application uses RFA's OMM Consumer model to retrieve level 1 data (MarketPrice) from the back-end server infrastructure.

The QuoteList can also be run as applet application. The applet script is quotelist.html file in /Examples/applet/ folder. More details see {@linkplain com.reuters.rfa.example.applet applet example package}.

The user selects which RICs and fields to display in the table. The data is displayed as lists of quotes and their associated field values. The display is updated in real time as data and state change events are received for each record and field.

This application uses the {@linkplain com.reuters.rfa.example.framework.sub framework.sub} for context initialization.

The QuoteList application consists of several distinct components

  • The FieldSelector, which allows the user to select which fields should be retrieved for a record. It also lets the user add fields to a default list individually or by specifying a record that contains all of the fields to ask.
  • The Status interface and LoggedStatusBar, which display and store status from services and item events.
  • ServiceSelector, which lets the user select from available record services. The table rendering components, RecordTableModel, RecordRenderer, and FieldRenderer, which display data in a table.
  • The QuoteList and QuoteListDisplay, which initialize the example and bring the GUI components together.

For GUI, The following command line configuration parameters are currently available.

  • help - Display help information and exit. The default is false.
  • session - Consumer session. The default is myNamespace::mySession.
  • serviceName - Service name to request the preload items. The default is "". If this is set to a service name, that service name must be available before requests are allowed. If this is not specified, and itemNames is specified, then the items will be requested from the first service in the directory.
  • itemNames - List of items to preload separated by ','. The default is an empty String.
  • fieldNames - List of field names to display separated by ','. The default is "DSPLY_NAME,TRDPRC_1,NETCHNG_1,ACVOL_1".
  • fontSize - Font size to use in the display. The default is 10.
  • font - Font name or logical font name. The default is the logical font name "Dialog".
  • user - DACS username for login. The default is the value from "user.name" property or "guest".
  • position - DACS position for login. The default is local host or "1.1.1.1/net".
  • application - DACS application ID for login. The default is "256".
  • fileDictionary - load dictionary from file. The default is false.
  • enumType - enumtype.def filename. The default is "/var/triarch/enumtype.def".
  • debug - enable debug tracing. The default is false.

The following parameter can be used if "type" is "OMM".

  • rdmFieldDictionary - RDMFieldDictionary filename. The default is "/var/triarch/RDMFieldDictionary".

The QuoteList application can be run with the following commands:

  • To retrieve MarketPrice items (using OMM consumer model)
    java com.reuters.rfa.example.quotelist.QuoteList -session myNamespace::rsslSession -itemName "RTR.L, IBM.N" -serviceName DIRECT_FEED
  • To run QuoteList as applet application
    Before running, an applet Jar need to be build. See {@linkplain com.reuters.rfa.example.applet applet}.
    Click quotelist.html to run with default parameters.
