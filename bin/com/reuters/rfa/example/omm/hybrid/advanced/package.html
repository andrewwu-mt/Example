This section contains an advanced hybrid application that is both an OMM consumer and an OMM provider.

The {@link com.reuters.rfa.example.omm.hybrid.advanced.AdvancedHybridDemo AdvancedHybridDemo} demonstrates how to write an application that is both an OMM consumer and an OMM provider. The OMM provider side of AdvancedHybridDemo listens to a request, and passes the request to the OMM consumer side of AdvancedHybridDemo. The OMM consumer side will create {@link com.reuters.rfa.session.omm.OMMConsumer OMMConsumer} everytime it receives a {@link com.reuters.rfa.rdm.RDMMsgTypes#LOGIN LOGIN} request.

The AdvancedHybridDemo will make {@link com.reuters.rfa.rdm.RDMMsgTypes#DIRECTORY DIRECTORY} upon receiving a LOGIN request to load data dictionaries for each service provided by the source application. While the dictionaries are loading, AdvancedHybridDemo puts any item request it receives into a pending state and sends them to the source application after the dictionaries load.

AdvancedHybridDemo also manages item groups. It will fanout group status and forward it to the client application.

AdvancedHybridDemo supports the following command line arguments:

  • runTime: Sets the number of seconds that the application runs. The default is 600.
  • session: Session must contain at least two connections (at least one must be RSSL and another must be RSSL_PROV). The default is myNamespace::hybridSession.
  • rdmFieldDictionary: Sets the RDMField dictionary name and location. The default is /var/triarch/RDMFieldDictionary.
  • enumType: Sets the RDMEnum dictionary name and location. The default is /var/triarch/enumType.def.
  • useReencoder: Sets whether to re-encode messages before passing them through the source/sink application. The default is true.
  • useEncodeData: Enables {@link com.reuters.rfa.example.omm.hybrid.OMMMsgReencoder#setEncodeData(boolean) useEncodeData} in {@link com.reuters.rfa.example.omm.hybrid.OMMMsgReencoder OMMMsgReencoder} when re-encoding a message. The default is false.
  • useEncodeString: Enables {@link com.reuters.rfa.example.omm.hybrid.OMMMsgReencoder#setEncodeString(boolean) useEncodeString} in {@link com.reuters.rfa.example.omm.hybrid.OMMMsgReencoder OMMMsgReencoder} when re-encoding a message. The default is false.
  • usePreEncodeDataDefs: Enables {@link com.reuters.rfa.example.omm.hybrid.OMMMsgReencoder#setPreEncodeDataDefs(boolean) usePreEncodeDataDefs} in {@link com.reuters.rfa.example.omm.hybrid.OMMMsgReencoder OMMMsgReencoder} when re-encoding a message. The default is false.

You can run AdvancedHybridDemo as an application by using the following command:

    java com.reuters.rfa.example.omm.hybrid.advanced.AdvancedHybridDemo
