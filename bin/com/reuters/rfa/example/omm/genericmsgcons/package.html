StarterConsumer_GenericMsg demonstrates subscribing to a 'custom' Domain Model to send and receive generic messages using RFA's OMM Consumer model. It can connect ONLY to the example application OMMGenericMsgProviderDemo {@link com.reuters.rfa.example.omm.genericmsgprov StarterProvider_GenericMsg}.

This application does the following:
  • Registers for Login
  • Handles the Login response
  • Registers/subscribes for one item through RFA
  • Initializes and continues a synchronous bidirectional conversation using generic messages with the example application OMMGenericMsgProviderDemo.
  • Use {@link com.reuters.rfa.example.utility.GenericOMMParser GenericOMMParser} to parse {@link com.reuters.rfa.omm.OMMMsg OMMMsg}

The StarterConsumer application currently supports the following command line arguments.

  • debug - enable detailed logging for RFA. The default is false.
  • session - name of consumer session to acquire. Its connection list should include one RSSL connection to OMMGenericMsgProvDemo. The default is RSSLNamespace::localConsumerSession.
  • user - DACS username for Login. The default is value of  user.name system property)
  • position - DACS position for login for Login. The default is IP address/hostname)
  • application - DACS application ID for Login. The default is 256.
  • runTime - How long (in seconds) application should run before exiting. The default is 600.

You can run this example as an application with the following commands:

    java com.reuters.rfa.example.omm.genericmsgcons.StarterConsumer_GenericMsg
