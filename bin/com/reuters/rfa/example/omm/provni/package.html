StarterProvider_NonInteractive is a simple provider example to demonstrate how to publish Reuters Domain Model data using the Open Message Model. This application is single-threaded client application. The applications connects and logs in to a ADH or P2PS POP. A TimerIntSpec is used to wake up the dispatch thread so it can send updates. Currently, the StarterProvider_NonInteractive application provides Service Directories and Market Price. It can be easily extended to provide other message model types.

StarterProvider_NonInteractive provides examples of how to:
  • Create a Session and an EventQueue
  • Create an OMMProvider event source
  • Register a Login request message
  • Dispatch events from an EventQueue
  • Cleanup a Session

The StarterProvider_NonInteractive application currently supports the following command line parameters.

  • debug - enable detailed logging for RFA (default is "false")
  • provSession - name of Provider session. (default is "myNamespace::provSession")
  • serviceName - Service name for the SrcDirectory response. (default is "DIRECT_FEED")
  • itemName - List of items to open separated by ','. (default is "TRI.N") Please note that ',' is a valid character for RIC name. The application need to be modified if RIC names have ','.
  • user - DACS username for Login (default is value of  user.name system property)
  • position - DACS position for login for Login (default is IP address/hostname)
  • application - DACS application ID for Login (default is "256")
  • updateInterval - Update interval in seconds. (default is "2")
  • updateRate - Update rate per interval. (default is "2")
  • sendStatus - Send Status Messages (default is "false")
  • serviceInfoOnUpdates - The serviceInfo to be set in attribInfo on update messages. The serviceInfo can either be a serviceID, serviceName or nothing. e.g. 1, DIRECT_FEED etc (default is ""; i.e. Updates do not contain serviceInfo)
  • dacsServiceId - The serviceID to generate DACS lock for PE(s);
    This can be obtained from Open DACS or a DACS administrator ( default is "1" )
  • refreshPE - Comma separated PEs for refresh messages; This requires dacsServiceId (default is ""; example 6566, 451)
  • updatePE - Comma separated PEs for update messages; This requires dacsServiceId (default is ""; example 6566, 451)
  • statusPE - Comma separated PEs for status messages; This requires dacsServiceId (default is ""; example 6566, 451)
  • runTime - How long (in seconds) application should run before exiting (default is "600")

The example can be run as an application with the following commands:

java com.reuters.rfa.example.omm.provni.StarterProvider_NonInteractive -serviceName DIRECT_FEED
