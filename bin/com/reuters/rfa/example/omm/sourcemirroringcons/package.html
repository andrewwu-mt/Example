StarterConsumer_SourceMirroring subscribes to one or more items using RFA's Generic Subscription model.

Package: com.reuters.rfa.example.omm.sourcemirroringcons

Source code location: $RFA/Examples/com/reuters/rfa/example/omm/sourcemirroringcons

StarterConsumer_SourceMirroring provides examples of how to:

  • Initialize and set command line options
  • Create a Session, EventQueue, Consumer Event Source, OMMEncoder, and OMMPool
  • Create LoginClient and DirectoryClient to handle Login / directory requests, responses, and generic messages.
    •  
  • Dispatch events from an EventQueue
    •  
  • Create GenericItemManager to handle the sending/receiving of generic item messages
  • Create Generic messages with Consumer Status Mode
  • Create Generic messages to start the generic message chain with provider
  • Cleanup a Session
Running the Example

This example uses the following command line parameters:

  • debug: Enables debug tracing. The default is false.
  • session: Sets the name of session to acquire. The default is RSSLNamespace::localConsumerSession.
  • user: Sets the DACS username for login. The default is the value of the user.name system property.
  • position: Sets the DACS position for login. The default is IP address/hostname.
  • application: Sets the DACS application ID for login. The default is 256.
  • runTime: Sets the duration (in seconds) for which the program runs before exiting. The default is 600.
  • serviceName: Specifies the service name from which to request messages/data. The default is DIRECT_FEED.
  • sourceMirroringMode: Sets the mode of this consumer. If this parameter is set to -1 the feature is off; set it to one of the following values: 0,1,2.

You can run this example as an application with the following command:

    java com.reuters.rfa.example.omm.sourcemirroringcons.StarterConsumer_SourceMirroring -sourceMirroringMode 0
