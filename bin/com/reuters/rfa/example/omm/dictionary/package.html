The DictionaryDemo application demonstrates how to download dictionaries from the network using RFA's OMM Consumer model. It can connect to an RFA OMMProvider / P2PS / RDFD using an RSSL connection that provides the dictionary domain.

The DictionaryDemo application:
  1. Registers for login and processes the login response.
  2. Registers for directory to get a list of dictionaries available for each service.
  3. Downloads full dictionaries from the network.
    Note: This application downloads dictionaries only from the first available service.
  4. Provides an option to read dictionary from file.
  5. If dictionaries from local files are available, it requests dictionaries {@linkplain com.reuters.rfa.rdm.RDMDictionary.Filter#INFO info} from the network and compares the version and ID.
    • If the dictionary version or ID is different, it downloads {@linkplain com.reuters.rfa.rdm.RDMDictionary.Filter#NORMAL full} the dictionary from the network.
    • If the dictionary version and ID are identical, it reads the dictionary from file.
      For example, if the field dictionary from the network and the local file have the same version and ID, but the enumeration dictionary versions are different, the application will read the field dictionary from file and download the enumeration dictionary from the network.
    • If both field and enumeration dictionaries from network and local files have the same version and ID, DictionaryDemo uses local files and does not display output.
  6. Display or dump to files after receiving the field and enumeration dictionaries from the network.
  7. Clean up and exit the program.

The DictionaryDemo application currently supports the following command line options.

  • session: Sets the name of the consumer session to acquire. Its connection list should include at least one RSSL connection. By default, DictionaryDemo uses myNamespace::mySession.
  • user: Sets the DACS username for Login. The default value is the value of user.name system property.
  • position: Sets the DACS position for Login. By default, DictionaryDemo uses IP address/hostname.
  • application: Sets the DACS application ID for Login. By default, DictionaryDemo uses 256.
  • rdmFieldDictionary: (Optional) Specifies the RDMFieldDictionary path and filename. By default, DictionaryDemo uses /var/triarch/RDMFieldDictionary.
  • enumType: (Optional) Specifies the enumtype.def path and filename. By default, DictionaryDemo uses /var/triarch/enumtype.def.
  • outputDir: Name of directory to store dump files of dictionaries from network. By default, DictionaryDemo uses stdout.

You can use the DictionaryDemo example in the following ways:

  • Download dictionaries from the network and print to stdout:
      java com.reuters.rfa.example.omm.dictionary.DictionaryDemo -session myNS::rsslSession
  • Download dictionaries from the network and dump the output to files:
      java com.reuters.rfa.example.omm.dictionary.DictionaryDemo -session myNS::rsslSession -dumpFiles true
  • Provide local dictionary files and let DictionaryDemo compare version and ID with downloaded dictionaries:
        java com.reuters.rfa.example.omm.dictionary.DictionaryDemo -session myNS::rsslSession -rdmFieldDictionary \Var\Triarch\RDMFieldDictionary -enumType \Var\Triarch\enumtype.def
