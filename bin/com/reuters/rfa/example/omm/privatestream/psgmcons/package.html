StarterConsumer_PrivateStreamGenericMsg demonstrates how to subscribe to a 'custom' Domain Model using a Private Stream to send and receive generic messages using RFA's OMM Consumer model. It can ONLY connect to the example application StarterProvider_PrivateStreamGenericMsg {@link com.reuters.rfa.example.omm.privatestream.psgmprov.StarterProvider_PrivateStreamGenericMsg StarterProvider_PrivateStreamGenericMsg}.

This application does the following:
  • Register for Login
  • Handle Login response
  • Request and handle a Directory
  • Register/subscribe one item through RFA using a Private Stream
  • Initialize and continue a synchronous bidirectional conversation using generic messages with the example application PrivateStrmGenMsgProvDemo.
  • Use {@link com.reuters.rfa.example.utility.GenericOMMParser GenericOMMParser} to parse {@link com.reuters.rfa.omm.OMMMsg OMMMsg}

The PrivateStrmGenMsgConsDemo application currently supports the following command line arguments.

  • debug - enable detailed logging for RFA (default is "false")
  • session - name of consumer session to acquire. Its connection list should include one RSSL connection to StarterProvider_PrivateStreamGenericMsg.  (default is "myNamespace::mySession")
  • user - DACS username for Login (default is value of  user.name system property)
  • position - DACS position for login for Login (default is IP address/hostname)
  • application - DACS application ID for Login (default is "256")
  • runTime - How long (in seconds) application should run before exiting (default is "600")
  • messageInterval - Time in seconds between each generic message exchange with responding peer (default is "5")

The example can be run as an application with the following commands:

java com.reuters.rfa.example.omm.privatestream.psgmcons.StarterConsumer_PrivateStreamGenericMsg
