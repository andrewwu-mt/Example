StarterConsumer_Post demonstrates how to subscribe and post to a provider application. It can connect to an RFA OMMProvider / P2PS / RDFD using an RSSL connection.

This application does the following:
  • Registers for Login
  • Handles Login response
  • Registers for item(s) request with the {@linkplain com.reuters.rfa.rdm.RDMMsgTypes#MARKET_PRICE MARKET_PRICE} model
  • Uses {@link com.reuters.rfa.example.utility.GenericOMMParser GenericOMMParser} to parse {@link com.reuters.rfa.omm.OMMMsg OMMMsg}
  • Handles {@link com.reuters.rfa.omm.OMMMsg OMMMsg} received via {@link com.reuters.rfa.session.omm.OMMItemEvent OMMtemEvent}
  • Identifies if the posting feature is supported.
  • Reads an input file with post details. Based on the input, it posts OMM Data or OMMMsg to a provider application.
  • Processes Ack messages received from a provider application.

The StarterConsumer_Post application supports the following command line arguments:

  • debug: Enables detailed logging for RFA. The default is false.
  • session: Sets the name of the consumer session to acquire. Its connection list should include at least one RSSL connection. The default is myNamespace::mySession.
  • serviceName: Sets the name of the source directory to use. The default is DIRECT_FEED.
  • itemName: Sets a comma-delimited list of items. The default is TRI.N.
    Note: Commas (,)are valid characters in a RIC name. If any RIC names have a comma, you must modify the application.
  • attribInfoInUpdates: Sets the example to tell the provider to send OMMAttribInfo with update and status messages. The default is false.
  • user: Sets the DACS username for Login. The default is the value of the user.name system property.
  • position: Sets the DACS position for Login. The default is IP address/hostname.
  • application: Sets the DACS application ID for Login. The default is 256.
  • rdmFieldDictionary: Sets the RDM Field dictionary name and location. The default is /var/triarch/RDMFieldDictionary.
  • enumType: Sets the enum dictionary name and location. . The default is /var/triarch/enumtype.def.
  • runTime: Sets the duration (in seconds) for which the application runs before exiting. The default is 600.
  • openItemStreams: Sets the example to open Items Streams by sending item requests. The default is true.
  • postInputFileName: Specifies the name of the input file with items to be posted. The file has all information necessary to do a post . The default is /postInput.txt.
  • dumpPost: Dump the post information read from the file to the console. The default is false.
  • sendPostAfterItemOpen: Send OnStream/OffStream Posts after all items are open. Otherwise it is sent when the login is granted. The default is true.

The example can be run as an application with the following commands:

    com.reuters.rfa.example.omm.postingConsumer.StarterConsumer_Post -itemName IBM.N,MSFT.O -serviceName DIRECT_FEED -postInputFileName C:\post_input.txt -dumpPost false -openItemStreams true - sendPostAfterItemOpen true

Note: An OMM consumer application is not required make a request for directory.  As a result this example does not request a DIRECTORY and the user needs to know the service name that is available
